## 오답노트
- `changeBeads()`에서 `endIdx==0`인 경우 처리를 계속 미스한다.
  - for문을 나오고의 연산인 만큼 endIdx의 제한 범위를 건너뛴다는 것을 잊지말아야한다.
  - 앞으로... for문 바깥에서 별도로 처리할 때는 for문을 거치지 않는 경우도 생각을 해보자.
- 그 외의 오답노트는 `Solution_bj_21611_마법사상어와블리자드.java`를 참고하자.
